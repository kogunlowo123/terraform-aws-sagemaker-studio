// terraform-aws-sagemaker-studio

AWS SageMaker Studio domain with JupyterServer, KernelGateway, VPC-only mode, lifecycle configurations, user profiles, spaces, and custom images.

Terraform ≥ 1.5.0 AWS ≥ 5.20.0 MIT License

Executive Summary

This module provisions an AWS SageMaker Studio domain with comprehensive configuration for data science teams. It creates a Studio domain in VPC-only mode by default, manages user profiles with per-user execution roles, shared collaboration spaces, lifecycle configurations for startup scripts, and custom SageMaker images. Networking is secured via a dedicated security group for NFS/EFS access, with optional KMS encryption for data at rest. An IAM execution role is auto-provisioned with permissions scoped to S3, ECR, CloudWatch, and SageMaker APIs.

Security Controls

VPC-Only Network Access

Default app_network_access_type is VpcOnly, blocking public internet access from Studio

KMS Encryption

Optional KMS key ARN for encrypting EFS and Studio data at rest

Security Groups

Dedicated SG for NFS/EFS with additional SG support via security_group_ids

IAM Auth / SSO

Validated auth_mode supporting IAM or AWS SSO federation

EFS Encryption

enable_efs_encryption defaults to true for encrypted home directories

Per-User Roles

Each user profile can have a dedicated execution role for least-privilege access

Production Readiness Checklist

  • Set app_network_access_type = "VpcOnly"
  • Provide kms_key_arn for data at rest encryption
  • Use SSO auth mode for enterprise federated access
  • Configure lifecycle configs to enforce security packages on startup
  • Assign per-user execution roles with scoped S3 bucket access
  • Deploy in 2+ AZ subnets for high availability
  • Lock down security groups to only allow NFS traffic from within VPC
  • Tag all resources with Environment, Team, and CostCenter

Deployment

module "sagemaker_studio" {
  source = "terraform-aws-sagemaker-studio"

  domain_name = "my-studio-domain"
  vpc_id      = "vpc-0123456789abcdef0"
  subnet_ids  = ["subnet-0123456789abcdef0"]

  user_profiles = {
    data_scientist = {
      name = "data-scientist"
      tags = { Role = "DataScientist" }
    }
  }

  tags = { Environment = "dev" }
}
